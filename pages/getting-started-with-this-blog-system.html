Getting Started with This Blog System

This blog system is designed to be simple, fast, and easy to use. Here's how to get started and create your first blog post.

What You Need

  • Node.js (version 14 or higher)
  • npm (comes with Node.js)
  • A text editor (VS Code, Sublime Text, etc.)

Quick Setup

  1. Install dependencies:

    npm install

  2. Build your blog:

    npm run build

  3. View your blog:

    npm run serve

That's it! Your blog is now running at http://localhost:8000.

Creating Your First Post

Step 1: Create a New Markdown File

Create a new file in the posts/ directory with a .md extension. For example: posts/my-first-post.md

Step 2: Add Frontmatter

Every post needs metadata at the top:

---
title: My First Blog Post
date: December 20, 2024
readTime: 3 min read
summary: This is a brief description of my post.
---

Step 3: Write Your Content

Now write your blog post using standard Markdown:

# My First Blog Post

Welcome to my blog! This is my first post using this amazing system.

## Why I Love This System

- **Simple**: Just write Markdown files
- **Fast**: No heavy frameworks or databases
- **Beautiful**: Clean, responsive design
- **Flexible**: Easy to customize and extend

Tips for Great Posts

  • Keep summaries short - 1-2 sentences max
  • Use clear headings - Structure your content well
  • Include examples - Code, images, or diagrams
  • Write for your audience - Keep language accessible
  • Proofread - Check for typos and clarity

Customization

The system is designed to be easily customizable:

  • Colors: Edit the CSS in index.html
  • Fonts: Change the Google Fonts link
  • Layout: Modify the HTML structure
  • Features: Extend the build script

Deployment

Since this generates a single HTML file, you can deploy it anywhere:

  • GitHub Pages: Free hosting for public repositories
  • Netlify: Drag-and-drop deployment
  • Vercel: Automatic deployments from Git
  • Any web server: Upload the HTML file

Need Help?

If you run into any issues:

  1. Check that Node.js is installed correctly
  2. Make sure all dependencies are installed (npm install)
  3. Verify your Markdown syntax is correct
  4. Check the console for any error messages
